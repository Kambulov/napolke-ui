## Начало работы c Napolke UI

#### Добро пожаловать в документацию

Napolke UI позволяет создавать красивые, современные и быстрые веб-сайты/приложения независимо от вашего опыта в дизайне, созданные с <a href="https://reactjs.org/" rel="noreferrer" target="_blank">React.js</a>.

### Настройка

Чтобы Napolke UI работал корректно, необходимо подключить NapolkeProvider в корне вашего приложения.

<Spacer h={1} />

#### React

Перейдите в корневую директорию вашего приложения и выполните следующее:

```tsx
// NAME: /src/app.js
import * as React from 'react'
import { NapolkeProvider, CssBaseline } from '@napolke-ui/core'

function App({ Component }) {
  return (
    <NapolkeProvider>
			<CssBaseline/> // --> Базовый стиль компонентов
      <Component />
    </NapolkeProvider>
  )
}
```

<Spacer h={2} />

<Spacer h={2} />

#### Next.js

1. Перейдите в файл pages/_app.js или pages/_app.tsx (создайте его, если он не существует), и добавьте следующий код:

```tsx
// NAME: pages/_app.ts
import { NapolkeProvider, CssBaseline } from '@napolke-ui/core'

function MyApp({ Component, pageProps }) {
  return (
    <NapolkeProvider>
			<CssBaseline/>
      <Component {...pageProps} />
    </NapolkeProvider>
  )
}

export default MyApp
```

2. Перейдите в файл  `pages/_document.js` или `pages/_document.tsx` (создайте его, если он не существует), и добавьте следующий код:

```jsx
// NAME: pages/_document.tsx
import React from 'react'
import Document, { Html, Head, Main, NextScript } from 'next/document'
import { CssBaseline } from '@napolke-ui/core'

class MyDocument extends Document {
  static async getInitialProps(ctx) {
    const initialProps = await Document.getInitialProps(ctx)
    return {
      ...initialProps,
      styles: React.Children.toArray([initialProps.styles])
    }
  }

  render() {
    return (
      <Html lang="кг">
        <Head>{CssBaseline.flush()}</Head>
        <body>
          <Main />
          <NextScript />
        </body>
      </Html>
    )
  }
}

export default MyDocument
```

>Для получения дополнительной информации о SSR с использованием Next.js и Napolke UI обратитесь к [документации]().

### Как использовать компоненты Napolke UI

После установки Napolke UI вы можете использовать любой из компонентов следующим образом.
Napolke UI использует оптимизацию дерева зависимостей (tree-shaking), поэтому неиспользуемые модули не будут включены в пакет во время процесса сборки, и
каждый компонент экспортируется отдельно.

```tsx
import { Button } from '@napolke-ui/core'

const Component = () => <Button>Отправить</Button>
```
